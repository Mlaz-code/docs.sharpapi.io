# Account Info

Get information about your account, API key, and usage.

## Endpoints

```
GET /api/v1/me
GET /api/v1/me/usage
GET /api/v1/me/preferences
```

## Account Details

Get your account and API key information.

```
GET /api/v1/me
```

## Usage Statistics

Get your API usage statistics.

```
GET /api/v1/me/usage
```

### Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `period` | string | `today` | Time period: `today`, `week`, `month`, `all` |

### Example Request

```bash
curl -X GET "https://api.sharpapi.io/api/v1/me/usage?period=month" \
  -H "X-API-Key: YOUR_API_KEY"
```

## Account Schema

| Field | Type | Description |
|-------|------|-------------|
| `keyId` | string | Your API key identifier |
| `tier` | string | `free`, `hobby`, `pro`, `sharp`, `enterprise` |
| `status` | string | `active`, `suspended`, `expired` |
| `createdAt` | string | When your account was created |
| `limits.requestsPerMinute` | number | Your rate limit |
| `limits.remaining` | number | Requests remaining this minute |
| `limits.resetsAt` | string | When the rate limit resets |
| `streaming.enabled` | boolean | Whether streaming is enabled |
| `streaming.maxConcurrent` | number | Max concurrent streams |
| `streaming.activeStreams` | number | Currently active streams |
| `features` | object | Feature access by tier |
| `sportsbooks.allowed` | number | Number of books you can access |
| `sportsbooks.list` | string[] | Sportsbooks you can access |

## Tier Information

| Tier | Requests/Min | Streams | Features |
|------|--------------|---------|----------|
| Free | 1 | 0 | Odds only |
| Hobby | 120 | 10* | Odds |
| Pro | 300 | 10* | + EV, Arb, Middles |
| Sharp | 2,000 | 10* | All features |
| Enterprise | Custom | Custom | All + SLA |

<Callout type="info">
*Streaming requires WebSocket add-on ($99/mo)
</Callout>

## Best Practices

1. **Monitor your usage** - Check `/me/usage` regularly to avoid hitting limits
2. **Track rate limit headers** - Use response headers for real-time limit tracking
3. **Set up alerts** - Enable notifications for rate limit warnings
4. **Upgrade before limits** - If you're consistently hitting limits, consider upgrading

<Callout type="warning">
If your account status is `suspended`, contact support to resolve any issues before making API requests.
</Callout>
