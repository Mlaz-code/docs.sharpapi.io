# Sportsbooks

Get list of available sportsbooks and their current status.

## Endpoints

```
GET /api/v1/sportsbooks
GET /api/v1/sportsbooks/{bookId}
```

<Callout type="info">
These endpoints are accessible without authentication at 10 requests/minute. Authenticated requests receive higher rate limits.
</Callout>

## List All Sportsbooks

### Example Request

<Tabs items={['cURL', 'JavaScript', 'Python']}>
  <Tabs.Tab>
```bash
curl -X GET "https://api.sharpapi.io/api/v1/sportsbooks" \
  -H "X-API-Key: YOUR_API_KEY"
```
  </Tabs.Tab>
  <Tabs.Tab>
```javascript
const response = await fetch(
  'https://api.sharpapi.io/api/v1/sportsbooks',
  { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
);
const { data } = await response.json();

const sharpBooks = data.filter(b => b.isSharp);
console.log('Sharp books:', sharpBooks.map(b => b.displayName));
```
  </Tabs.Tab>
  <Tabs.Tab>
```python
import requests

response = requests.get(
    'https://api.sharpapi.io/api/v1/sportsbooks',
    headers={'X-API-Key': 'YOUR_API_KEY'}
)
books = response.json()['data']
sharp_books = [b for b in books if b.get('isSharp')]
print(f"Sharp books: {[b['displayName'] for b in sharp_books]}")
```

## Sportsbook Schema

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier (use in API filters) |
| `name` | string | Internal name |
| `displayName` | string | Human-readable name |
| `hasLiveOdds` | boolean | Supports live/in-play odds |
| `hasPlayerProps` | boolean | Supports player prop markets |
| `region` | string[] | Available regions (`US`, `UK`, `EU`, `GLOBAL`) |
| `isSharp` | boolean | Whether book is considered sharp |
| `status` | string | `active` or `inactive` |
| `oddsCount` | number | Number of odds currently available |
| `lastUpdate` | string | ISO 8601 timestamp of last data update |
| `sports` | string[] | Supported sports (single book only) |
| `markets` | string[] | Supported market types (single book only) |

## Sharp vs Soft Books

### Sharp Books
- Lower vig (juice)
- Better odds accuracy
- Used as reference for +EV calculations
- Examples: Pinnacle, Circa

### Soft Books
- Higher vig
- More promotions and bonuses
- Where +EV opportunities are found
- Examples: DraftKings, FanDuel, BetMGM

<Callout type="info">
Sharp books (like Pinnacle) set efficient lines. Soft books often lag behind, creating +EV opportunities.
</Callout>
