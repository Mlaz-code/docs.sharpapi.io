# Sportsbooks

Get list of available sportsbooks and their current status.

```
GET /api/v1/books
```

## Response

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | Whether the request succeeded |
| `timestamp` | string | ISO 8601 timestamp |
| `count` | number | Total number of sportsbooks |
| `active` | number | Number of sportsbooks currently providing data |
| `books` | array | Array of sportsbook configurations |

### Example Request


```bash cURL
curl -X GET "https://sharpapi.io/api/v1/books" \
  -H "X-API-Key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  'https://sharpapi.io/api/v1/books',
  { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
);
const { books } = await response.json();

const sharpBooks = books.filter(b => b.isSharp);
console.log('Sharp books:', sharpBooks.map(b => b.displayName));
```

## Book Schema

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier (use in API filters) |
| `displayName` | string | Human-readable name |
| `hasLiveOdds` | boolean | Supports live/in-play odds |
| `hasPlayerProps` | boolean | Supports player prop markets |
| `region` | string[] | Available regions |
| `isSharp` | boolean | Whether book is considered sharp |
| `status` | string | `active` or `inactive` - whether currently providing data |
| `oddsCount` | number | Number of odds currently available |
| `lastUpdate` | string? | ISO 8601 timestamp of last data update (null if inactive) |

## Sharp vs Soft Books


  ### Sharp Books

    - Lower vig (juice)
    - Better odds accuracy
    - Used as reference for +EV
    - Examples: Pinnacle, Circa
  
  ### Soft Books

    - Higher vig
    - More promotions
    - Where +EV opportunities exist
    - Examples: DraftKings, FanDuel
  


> **Tip:**
  Sharp books (like Pinnacle) set efficient lines. Soft books often lag behind, creating +EV opportunities.
